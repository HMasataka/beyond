# ADR-0002: 依存は Nix、起動するサービスは Docker Compose で管理

- ステータス: Accepted
- 日付: 2026-06-20

## 背景

開発と実行に必要なものは、性質の異なる 2 種類に分かれる。

- 依存: 言語処理系、ビルドツール、CLI、ライブラリなど、開発者の環境に用意する道具。
- 起動するサービス: データベースやキャッシュ、アプリ本体など、実行時にプロセスとして立ち上げるもの。

この 2 種類を 1 つの仕組みでまとめて扱うと、環境ごとに差が出たり、再現の手順が曖昧になったりする。
それぞれに適した道具で分けて管理したい。

## 決定

依存とサービス起動を別の道具に割り当てる。

- 依存は基本的に Nix で管理する。開発環境に必要な道具のバージョンを固定し、誰が用意しても同じ環境になるようにする。
- 起動するサービスは Docker Compose で管理する。定義は `compose.yaml` に書く（`docker-compose.yml` などの別名は使わない）。

## 結果/影響

- 開発環境の道具は Nix で固定され、マシンをまたいでも同じ依存が揃う。
- 実行時に立ち上げるサービスは `compose.yaml` の一箇所に集約され、起動方法が明確になる。
- 依存とサービスの責務が分かれ、どちらをどこで直すかが一目で分かる。

## 注意点

- サービス定義の追加や変更は `compose.yaml` に対して行う。ファイル名は `compose.yaml` に統一する。
