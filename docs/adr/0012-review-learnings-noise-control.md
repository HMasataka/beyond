# 0012. レビュー知見のノイズ対策（承認ゲートと棚卸し）

- ステータス: Accepted
- 日付: 2026-06-24

## コンテキスト

ADR 0011 で会話ログからの自動抽出を入れた結果、台帳に入る知見が増える。
自動抽出は本体エージェントの判断に依存し、玉石混交になりうる。質の低い知見がそのまま `active` で混ざると、卒業しきい値の算入が歪み、台帳が雑音で汚れる。

ADR 0011 の「結果」は、このノイズ対策（承認待ち・定期 prune）を本 ADR の対象外とし、別の決定で扱うとした。本 ADR がそれを回収する。

制約は ADR 0010 / 0011 を引き継ぐ。

- status の遷移は人手のみで、自動遷移しない（ADR 0010・0011）。
- スクリプトは POSIX sh + awk のみで書き、jq / node / task / nix に依存しない。
- フックは薄く保ち、新規フックを増やすと codex 移植が複雑化する。

## 決定

ノイズ対策を「承認ゲート」と「棚卸し補助」の 2 本立てで入れる。

- status に `pending`（承認待ち）を加える。会話由来の自動抽出（extract）は `pending` で記録し、人が確認してから採用（`active`）か却下（`retired`）へ手で遷移する。PR 由来のレビュー指摘は人手の記録なので従来どおり `active` で入る。status の自動遷移はしない。
- `propose.sh` は卒業しきい値に `active` のみを算入する。これは既存実装（`status != "active"` を除外）で満たされる。`pending` は新たに除外されるが、改修は不要で、除外は既存ロジックで担保される。
- `capture.sh` に `--status <active|pending>`（既定 `active`）を加える。新規ファイル作成時の初期 status にだけ反映し、既存 slug への occurrence 追記では status に触れない（後方互換）。`promoted` / `retired` は capture では設定させない。人手遷移の方針を侵さないためで、不正値は die する。
- 棚卸し補助として読み取り専用の `prune.sh` を加える。台帳には一切書き込まず、棚卸し候補を 3 区分で一覧する。放置 pending（承認されず古い）・死蔵 active（卒業しきい値未到達で古い）・retired（廃止済み）を出す。`promoted`（卒業済み）は対象外。`--days N`（既定 90）で「古い」の境界を決める。壊れた台帳でクラッシュせず、該当ファイルを読み飛ばす。
- prune の日付計算は移植性を担保する。当日エポックから `cutoff_epoch` を作り、`date -u -d "@…"`（GNU）→ `date -u -r …`（BSD/macOS）の順で cutoff 日付を得る。どちらも無ければ古さ判定を degrade し、古さに依存する候補（放置 pending・死蔵 active）は出さず、retired のみ一覧する。日付比較は YYYY-MM-DD の辞書順でよい。

## 結果

- 自動抽出由来の知見が `pending` で隔離され、人の承認を経るまで卒業しきい値に算入されない。台帳の雑音が卒業判断に混ざらない。
- prune が放置・死蔵・廃止済みの知見を可視化し、棚卸しの起点になる。読み取り専用なので台帳を壊さない。
- 既存台帳（`status: active` のみ）・既存の capture 呼び出し・`propose.sh`・symlink は無改変で動く。`--status` 省略時は従来どおり `active` で作られる。
- 残る制約: 承認・retire・削除はすべて人手で、運用の規律が要る。prune は候補を出すだけで自動では片付けない。
- 自動削除・自動 retire をしない理由: status 遷移を人手のみとする ADR 0010 / 0011 の方針と整合させる。自動削除は判断の誤りで知見を失うリスクがあり、取り返しがつかない。prune を一覧に限れば、誤検出があっても人が無視するだけで済む。
- 却下した代替案:
  - prune が古い知見を自動で retire / 削除する。status を人手のみとする方針に反し、誤判定で知見を失う。一覧に留めれば誤検出のコストは無視で済むため、自動化は採らない。
  - pending の件数を `SessionStart` などのフックで自動通知する。新規フックは codex 移植を複雑化し、ADR 0011 の「フックを増やさない」方針に反する。承認は prune / approve の手動フローで足り、通知フックは採らない。
  - 近似重複の知見を自動マージする。類似判定には文脈理解が要り、決定的スクリプトでは誤マージのリスクが高い。本 ADR のスコープ外とする。
