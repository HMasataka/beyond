# 0009. AGENTS.md と skill 定義を正本とし各ツール用ファイルをシンボリックリンクにする

- ステータス: Accepted
- 日付: 2026-06-24

## コンテキスト

このリポジトリは複数の AI コーディングツールから扱う。
各ツールは読み込む規約ファイルが異なり、Claude Code は `CLAUDE.md` を、ほかのツールは別名のファイルを読む。
skill 定義も `.claude` や codex 用など、ツールごとに置き場が分かれる。

同じ内容を各ファイルに実体として重複させると、片方だけ更新したときに内容が乖離する。
更新漏れを防ぐ仕組みが要る。

## 決定

規約と skill 定義は一箇所を正本とし、ツールごとのファイルはそこへのシンボリックリンクにする。

- プロジェクト固有の規約は `AGENTS.md` を正本とし、`CLAUDE.md` は `AGENTS.md` へのシンボリックリンクにする。
- サブディレクトリに `AGENTS.md` を置いたら、同じディレクトリに `CLAUDE.md` をシンボリックリンクとして作る（`ln -s AGENTS.md CLAUDE.md`）。
- 規約の追加・更新は常に `AGENTS.md` に対して行う。Claude 専用ルールのつもりで `CLAUDE.md` を直接編集しない。
- skill も `.agents` を正本とし、`.claude` や codex 用はシンボリックリンクにする。
- `AGENTS.md` は目次として軽く保つ。各節には規範の要約と参照先リンク、参照先をいつ読むべきかのトリガーだけを置き、詳細は参照先のドキュメントに書く。

## 結果

- 規約と skill 定義の実体が一箇所に集まり、どのツールから読んでも同じ内容になる。更新漏れによる乖離が起きない。
- 読み込み先のファイルが増えても、シンボリックリンクを足すだけで対応できる。
- `AGENTS.md` を目次に保つことで、エージェントが読むコンテキスト量を抑えられる。
- 残る制約: シンボリックリンクを解決できない環境やツールでは実体を参照できない。Git はシンボリックリンクをそのまま記録するため、リポジトリを扱う側がシンボリックリンクに対応している前提になる。
- 却下した代替案: 各ファイルに実体を置き、内容を手動またはスクリプトで同期する。同期の手間が残り、片方だけ更新して乖離するリスクも消えないため採らない。
