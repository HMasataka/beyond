# 0003. タスク実行を Task で統一する

- ステータス: Accepted
- 日付: 2026-06-21

## コンテキスト

このリポジトリは Go の API と TypeScript の web を同居させており、開発で使う操作はビルド、テスト、lint、コード生成、コンテナ操作にまたがる。
これらのコマンドは引数や作業ディレクトリが異なり、覚えておくべき手順が増える。
共通の実行口がないと、手順が README や各人の記憶に散らばり、環境ごとに呼び方が変わる。

ADR 0002 で、依存は Nix、起動するサービスは Docker Compose と決めた。
一方で、それらを呼び出す日々のタスクをどこにまとめるかは決めていない。

タスクランナーの候補は次のとおり。

- **Make**：広く使われるが、タブに依存する構文で記述ミスを誘いやすく、名前空間や一覧表示の仕組みが弱い。
- **npm scripts**：web の package.json に集約することになり、Go を含むリポジトリ全体の入口としては言語が偏る。
- **素のシェルスクリプト**：自由度は高いが、利用できるタスクの一覧を示す仕組みがなく、発見性が低い。

## 決定

タスク実行を Task（go-task）に統一し、定義はリポジトリ直下の `Taskfile.yml` に書く。

ビルド、テスト、lint、コード生成、コンテナ操作を `Taskfile.yml` のタスクとして定義する。
ディレクトリをまたぐ操作は、タスクごとに作業ディレクトリを指定して表現する。

## 結果

`task --list-all` で利用できるタスクを一覧でき、入口が一箇所に集まる。
api と web のどちらの操作も同じ呼び方で実行でき、環境が変わっても手順が揃う。

タスク定義のファイル名は `Taskfile.yml` に統一する。
